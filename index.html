Project S AI / Automation

Python · Gemma 3 27B · Llama 3.3 70B · Qwen3-TTS · Stable Diffusion XL · FFmpeg · OpenRouter

I run four YouTube channels in the horror niche. After a while the manual production process got to be a lot, so I built a pipeline to handle it overnight. One command kicks it off and by morning there's a finished, upload-ready video sitting in the output folder.

The pipeline generates the script using a couple of different LLMs, synthesizes narration audio locally on my GPU using Qwen3-TTS, generates imagery through Stable Diffusion XL running in ComfyUI, and assembles everything into a video with FFmpeg using NVENC hardware encoding. The part that took the most work was the script quality loop: it evaluates drafts against a rubric and reruns until it hits a threshold, so the outputs don't feel robotic.

Battery Retrofit Proposal Tool Internal Tool

Python · ReportLab · PyPDF2 · openpyxl · Excel / VBA

At my previous job, sales reps selling battery add-ons to existing solar customers had to submit a request to a back-office team and wait for someone to manually put together a proposal PDF. It could take hours, and by the time the proposal arrived the customer had often moved on.

I built a tool to fix that. The rep opens an Excel workbook, fills in five fields, picks the battery product and financing type, and clicks a button. Python reads the calculated pricing and loan outputs from the workbook via openpyxl, overlays all the customer information onto a branded PDF template using ReportLab, and writes a finished proposal to the output folder. The whole process takes a few seconds. Financial logic stays in Excel where the finance team can update it without touching any code.
